### **Specification: Bilingual ePub/PDF/MD Translator CLI**

#### **1. Purpose & Scope**
A CLI tool to assist in translating books/documents by dividing the content into chunks for easier translation via LLM, and then combining the results into a bilingual (para-by-para) ePub file, emphasizing the accuracy of the content sequence.

#### **2. Core Components**
* **Source Parser:** This reads the source file and breaks it down into... `List[Paragraph]` (Object storing content and index)
* **Chunking Logic:**
* **Buffer-Aware:** Uses character count as a criterion (Limit: 5,000 or as specified)
* **Atomic:** Does not truncate paragraphs mid-line. If the next paragraph exceeds the limit, the chunk should end at the current paragraph.
* **Rule Engine:** Dictionary system (JSON/YAML) for storing key-value pairs for dynamic injection into the prompt.
* **Reconstructor:** System for combining files using strict indexing `[n]` to map English and Thai content together.

#### **3. Data Structure & Workflow**
* **Directory Structure:**
```text
/my-book-project/
├── config.yaml # Stores Prompt Template and Dictionary
├── source/ # Source file (epub/pdf/md)
├── chunks/ # .txt file generated by the CLI (with Prompt + Index)
├── translated/ # .txt file where the user places the translation (Filename must match chunks/)
└── output/ # Complete Bilingual ePub File
```

#### **4. The Prompt Template (Structure)**
```text
[System Instructions]
... instructions about bilingual translation ...
... keep [n] index ...

[Terminology Rules] (Only if keyword exists in chunk)
- {key}: {value}

[Source Text]
[1] {para_1}
[2] {para_2}
```

#### **5. Error Handling & Validation**
* **Mismatch Check:** If the number of indexes in the translation file does not match the original, the system will display an error and indicate the missing index number.
* **Format Integrity:** Before merging into an ePub, the system will automatically remove the index number `[n]` from the content of both languages.
